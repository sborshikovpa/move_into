## 1. Что такое структура данных?

**Структура данных** — это способ организации информации, чтобы её было удобно:

- хранить;
- находить;
- добавлять;
- удалять;
- изменять.

Представь комнату с одеждой:

- если одежда лежит одной кучей, найти носок сложно;
- если она разложена по подписанным ящикам, найти носок легко.

Данные остались теми же. Изменился только способ их организации.

---

# Как данные хранятся в памяти

## 2. Что такое оперативная память?

Оперативную память можно представить как длинный ряд пронумерованных ячеек:

```
Адрес:      1000   1004   1008   1012
Значение:     10     20     30     40
```

**Адрес** — это номер места в памяти.

Переменная может хранить само простое значение или ссылку на объект:

```
numbers = [10, 20, 30]
```

Упрощённо это выглядит так:

```
Переменная                 Память
numbers ───── ссылка ────→ [10, 20, 30]
```

Имя `numbers` и сам список — не одно и то же. Переменная помогает компьютеру найти объект в памяти.

# Основные структуры данных

## 3. Массив

Массив похож на ряд пронумерованных шкафчиков:

```
Индекс:       0       1       2       3
Значение:    10      20      30      40
```

Обращение к элементу происходит по индексу:

```
numbers = [10, 20, 30, 40]

print(numbers[2])  # 30
```

Индексы начинаются с нуля.

### Как массив хранится в памяти

В классическом массиве элементы располагаются подряд:

```
Адрес:      1000   1004   1008   1012
Значение:     10     20     30     40
Индекс:        0      1      2      3
```

Если каждый элемент занимает 4 байта, компьютер может вычислить его адрес:

```
адрес = начало массива + индекс × размер элемента
```

Для элемента с индексом `2`:

```
1000 + 2 × 4 = 1008
```

Компьютер сразу понимает, куда обратиться. Поэтому доступ по индексу занимает `O(1)`.

### Вставка в середину

Если вставить `15` после `10`:

```
Было:

[10][20][30][40]

Стало:

[10][15][20][30][40]
```

Элементы `20`, `30` и `40` придётся сдвинуть. Поэтому вставка в середину может занимать `O(n)`.

### Особенность Python

Список Python хранит подряд не сами объекты, а ссылки на них:

```
Список:
[ ссылка ][ ссылка ][ ссылка ]
     │         │         │
     ▼         ▼         ▼
    10       "кот"      True
```

Поэтому в одном списке могут находиться данные разных типов:

```
data = [10, "кот", True]
```

### Когда использовать массив или список

- нужно хранить последовательность элементов;
- важен порядок;
- нужен доступ по индексу;
- нужно перебрать все элементы.

## 4. Связный список

Связный список похож на поезд. Каждый вагон хранит:

1. значение;
2. ссылку на следующий вагон.

```
[10 | ссылка] → [20 | ссылка] → [30 | конец]
```

### Как он хранится в памяти

Узлы могут находиться в совершенно разных местах:

```
Адрес 1000               Адрес 8000
[10 | ссылка 8000] ────→ [20 | ссылка 3500]
                                │
                                ▼
                         Адрес 3500
                         [30 | конец]
```

Связи на рисунке — это ссылки, а не физическое соседство элементов.

### Поиск элемента

Чтобы найти третий элемент, нужно пройти предыдущие:

```
первый → второй → третий
```

Поэтому доступ по позиции занимает `O(n)`.

### Вставка элемента

Чтобы вставить `20` между `10` и `30`, не нужно двигать остальные данные:

```
Было:

[10] → [30]

Стало:

[10] → [20] → [30]
```

Нужно создать новый узел и изменить ссылки.

### Когда использовать

- элементы часто добавляются или удаляются;
- быстрый доступ по индексу не нужен;
- важно понять устройство других структур.

На практике готовые списки языка часто удобнее.

## 5. Стек

Стек работает по правилу:

> Последним пришёл — первым ушёл.

Представь стопку тарелок. Последнюю положенную тарелку снимут первой.

Это называется **LIFO**:

```
Last In, First Out
```

### Пример

```
stack = []

stack.append("первая")
stack.append("вторая")
stack.append("третья")

print(stack.pop())  # третья
```

### Как стек хранится в памяти

Стек можно реализовать с помощью массива:

```
Низ стека

┌────────┐
│   10   │
├────────┤
│   20   │
├────────┤
│   30   │ ← вершина
└────────┘
```

Добавление элемента:

```
push(40)

[10][20][30][40]
             ↑
          вершина
```

Удаление:

```
pop()

[10][20][30]
         ↑
      вершина
```

Работа происходит только с вершиной, поэтому добавление и удаление обычно занимают `O(1)`.

### Где используется стек

- отмена действий;
- история переходов;
- вызовы функций;
- проверка скобок;
- прохождение лабиринтов.

Например, при проверке скобок:

```
([ ])
```

Открывающие скобки кладутся в стек. Закрывающая скобка должна совпадать с последней открытой.

---

## 6. Очередь

Очередь работает как очередь в магазине:

> Первым пришёл — первым обслужен.

Это называется **FIFO**:

```
First In, First Out
```

### Пример

```
from collections import deque

queue = deque()

queue.append("Аня")
queue.append("Борис")
queue.append("Вика")

print(queue.popleft())  # Аня
```

### Как очередь хранится в памяти

Очередь можно хранить в массиве. Нужно помнить начало и конец:

```
[пусто][Аня][Борис][Вика][пусто]
          ↑             ↑
       начало          конец
```

После удаления Ани начало сдвигается:

```
[пусто][Аня][Борис][Вика][пусто]
               ↑        ↑
            начало     конец
```

Элементы не обязательно физически перемещать.

Иногда используется **кольцевой буфер**. После достижения конца массива запись продолжается с начала:

```
┌─────────────────────────┐
│ [Глеб][пусто][Аня][Вика]│
└─────────────────────────┘
```

В Python для очереди лучше использовать `deque`, потому что удаление первого элемента обычного списка выполняется медленнее.

### Где используется очередь

- обработка задач;
- печать документов;
- сообщения;
- запросы к серверу;
- поиск кратчайшего пути.

---

## 7. Словарь или хеш-таблица

Словарь похож на шкаф с подписанными ящиками:

```
"имя"     → "Антон"
"возраст" → 25
"город"   → "Москва"
```

В Python используется `dict`:

```
user = {
    "name": "Антон",
    "age": 25,
    "city": "Москва"
}

print(user["city"])  # Москва
```

Значение ищется не по номеру, а по **ключу**.

### Как словарь хранится в памяти

Внутри используется хеш-таблица:

```
Ячейка 0: [пусто]
Ячейка 1: ["city" → "Москва"]
Ячейка 2: [пусто]
Ячейка 3: ["name" → "Антон"]
Ячейка 4: ["age" → 25]
```

Когда выполняется:

```
user["name"]
```

происходит примерно следующее:

```
"name"
   ↓
хеш-функция
   ↓
число
   ↓
номер ячейки
   ↓
"Антон"
```

Компьютеру обычно не нужно перебирать весь словарь. Поэтому поиск в среднем занимает `O(1)`.

### Что такое коллизия?

Иногда разные ключи получают одинаковый номер ячейки:

```
"кот" → ячейка 3
"дом" → ячейка 3
```

Это называется **коллизией**.

Хеш-таблица умеет разрешать такие ситуации, например ищет другую свободную ячейку.

### Пример подсчёта слов

```
words = ["кот", "пёс", "кот"]
counts = {}

for word in words:
    counts[word] = counts.get(word, 0) + 1

print(counts)
# {'кот': 2, 'пёс': 1}
```

### Где используется словарь

- поиск пользователей по ID;
- настройки;
- подсчёт значений;
- кеширование;
- описание объектов.

---

## 8. Множество

Множество хранит только уникальные элементы:

```
numbers = {1, 2, 2, 3, 3, 3}

print(numbers)  # {1, 2, 3}
```

Повторяющиеся значения автоматически исчезают.

### Как множество хранится в памяти

Множество похоже на словарь без значений:

```
Словарь:

"name" → "Антон"

Множество:

"Антон"
```

Внутри также используется хеш-таблица:

```
Ячейка 0: [пусто]
Ячейка 1: ["Аня"]
Ячейка 2: [пусто]
Ячейка 3: ["Борис"]
Ячейка 4: ["Вика"]
```

Поэтому проверка наличия элемента в среднем занимает `O(1)`:

```
if "Аня" in students:
    print("Найдена")
```

### Пересечение множеств

```
python_students = {"Аня", "Борис", "Вика"}
java_students = {"Борис", "Вика", "Глеб"}

print(python_students & java_students)
# {'Борис', 'Вика'}
```

### Когда использовать

- нужно удалить повторы;
- нужно быстро проверить наличие;
- нужно сравнивать группы элементов.

---

## 9. Дерево

Дерево используется для хранения иерархических данных.

Например, папки компьютера:

```
Проект
├── Документы
│   ├── Отчёт
│   └── Договор
└── Фотографии
    ├── Отпуск
    └── Работа
```

Основные понятия:

- **корень** — главный элемент;
- **узел** — элемент дерева;
- **родитель** — узел выше;
- **ребёнок** — узел ниже;
- **лист** — узел без детей.

### Бинарное дерево поиска

У каждого узла может быть не больше двух детей:

```
        8
       / \
      3   10
     / \    \
    1   6    14
```

Меньшие значения находятся слева, большие — справа.

### Как дерево хранится в памяти

Узел может хранить:

- значение;
- ссылку на левого ребёнка;
- ссылку на правого ребёнка.

```
┌──────────────────────────┐
│ значение: 8              │
│ левый ребёнок: адрес A   │
│ правый ребёнок: адрес B  │
└──────────────────────────┘
```

Узлы могут находиться в разных местах памяти:

```
       [8]
      /   \
     ▼     ▼
   [3]    [10]
```

Дерево расходует дополнительную память на хранение ссылок.

### Где используется

- файловая система;
- HTML-страницы;
- меню;
- базы данных;
- поиск;
- игровое поведение.

---

## 10. Граф

Граф — это набор объектов и связей между ними:

```
Аня ─── Борис
 │        │
 Вика ── Глеб
```

Объекты называются **вершинами**, а связи — **рёбрами**.

Примеры:

- пользователи социальной сети — вершины;
- дружба — рёбра;
- города — вершины;
- дороги — рёбра.

### Список смежности

Граф можно хранить в виде словаря:

```
graph = {
    "Аня": ["Борис", "Вика"],
    "Борис": ["Аня", "Глеб"],
    "Вика": ["Аня"],
    "Глеб": ["Борис"]
}
```

В памяти это выглядит примерно так:

```
Аня   → [Борис, Вика]
Борис → [Аня, Глеб]
Вика  → [Аня]
Глеб  → [Борис]
```

Такой способ обычно экономно расходует память.

### Матрица смежности

Другой вариант:

```
        Аня  Борис  Вика  Глеб
Аня      0     1      1     0
Борис    1     0      0     1
Вика     1     0      0     0
Глеб     0     1      0     0
```

`1` означает, что связь есть, а `0` — что её нет.

Матрица позволяет быстро проверить связь, но может занимать много памяти.

### Где используются графы

- социальные сети;
- карты и навигаторы;
- интернет;
- рекомендации;
- зависимости между задачами.

# Сложность алгоритмов

## 11. Что означает `O(...)`?

Запись `O(...)` показывает, как увеличивается количество работы при росте объёма данных.

Это не точное время в секундах, а оценка скорости роста.

---

## `O(1)` — постоянное время

Количество элементов почти не влияет на операцию:

```
numbers[2]
user["name"]
```

Компьютер сразу обращается к нужному месту.

---

## `O(n)` — линейное время

Иногда нужно проверить все элементы:

```
for number in numbers:
    if number == 100:
        print("Нашли")
```

Если элементов станет в 10 раз больше, работы тоже станет примерно в 10 раз больше.

---

## `O(log n)` — каждый раз отбрасываем половину

Так работает бинарный поиск в отсортированном массиве:

```
1 000 000 элементов
500 000 элементов
250 000 элементов
125 000 элементов
...
```

Чтобы найти значение среди миллиона элементов, потребуется примерно 20 шагов.

---

## `O(n²)` — перебираем пары элементов

```
for first in numbers:
    for second in numbers:
        print(first, second)
```

Количество действий:

```
10 элементов    → примерно 100 действий
100 элементов   → примерно 10 000 действий
1000 элементов  → примерно 1 000 000 действий
```

На больших данных такие алгоритмы могут работать медленно.

# Стек и куча памяти

## 12. Стек памяти

Не нужно путать структуру данных **стек** с областью памяти, которая тоже называется стеком.

Стек памяти обычно хранит информацию о текущих вызовах функций:

```
def greet(name):
    message = "Привет, " + name
    return message
```

Упрощённо:

```
Стек вызовов:

┌────────────────────┐
│ функция greet      │
│ переменная name    │
│ переменная message │
└────────────────────┘
```

Когда функция заканчивается, её запись удаляется из стека вызовов.

---

## 13. Куча памяти

В куче обычно находятся динамически созданные объекты:

```
Переменные                 Куча

numbers ─────────────────→ [10, 20, 30]

user ────────────────────→ {"name": "Антон"}
```

Куча как область памяти и структура данных `heap` — разные понятия с одинаковым названием.

---

## 14. Управление памятью и сборка мусора

Рассмотрим Python:

```
numbers = [10, 20, 30]
numbers = None
```

Сначала переменная ссылается на список:

```
numbers ──→ [10, 20, 30]
```

После присваивания `None`:

```
numbers ──→ None

[10, 20, 30] ← объект больше не используется
```

Python может освободить память этого объекта.

Автоматическое удаление ненужных объектов связано с подсчётом ссылок и работой **сборщика мусора**.

|---|---|---|---|
|Массив|шкафчики с номерами|элементы или ссылки подряд|доступ по индексу|
|Связный список|поезд|отдельные узлы со ссылками|вставка и удаление|
|Стек|стопка тарелок|массив или связанные узлы|отмена, история|
|Очередь|очередь в магазине|кольцевой массив или узлы|обработка по порядку|
|Словарь|подписанные ящики|хеш-таблица|поиск по ключу|
|Множество|коллекция без повторов|хеш-таблица без значений|уникальные элементы|
|Дерево|папки компьютера|узлы со ссылками на детей|иерархические данные|
|Граф|карта дорог|списки или матрица связей|маршруты и отношения|

# Главное, что нужно запомнить

1. **Структура данных** — это способ организации информации.
2. Массив быстро получает элемент по индексу.
3. Связный список удобен для вставки и удаления узлов.
4. Стек работает по принципу «последним пришёл — первым ушёл».
5. Очередь работает по принципу «первым пришёл — первым ушёл».
6. Словарь быстро ищет значение по ключу.
7. Множество хранит уникальные элементы.
8. Дерево хранит иерархию.
9. Граф хранит объекты и связи между ними.
10. Скорость структуры зависит от того, как данные расположены в памяти.

> Не существует одной лучшей структуры данных. Для каждой задачи выбирается структура, с которой эту задачу проще и быстрее решить.
